## 🚀 Features

### 🩸 Core System
- **Complete Blood Donation Management** — End-to-end workflow from donor registration to blood unit tracking.
- **Blood Banking** — Real-time inventory management of blood units by type and availability.
- **Request & Handover System** — Streamlined processing of blood requests with a dedicated handover workflow for safe release of units.
- **Dual-Module System** — Separate, role-based interfaces for Administrators and regular users.

### 📍 Google Maps Integration
- **Blood Drive Locator** — Interactive map powered by Google Maps API to help users find nearby blood donation drives and partner blood banks.

### 📲 QR-Based Attendance System
- **Digital Member ID Scanning** — Admins can scan members' QR-coded digital IDs at donation venues for fast and accurate attendance logging.
- **Event Attendance Tracking** — Automated attendance records tied to donation events.

### 📊 Predictive Analytics
- **Forecasting Scripts** — Python-powered forecasting to anticipate blood supply needs and prevent shortages.

## ⚙️ Local Setup

1. **Clone the repository**
```bash
   git clone https://github.com/your-username/ServingHearts.git
```

2. **Move to XAMPP**
   Copy the project folder into `xampp/htdocs/`.

3. **Database Configuration**
   - Import the provided `.sql` file into your MySQL database via phpMyAdmin or the CLI.
   - Update `config.php` or `db.php` with your database credentials.

4. **Google Maps API**
   - Obtain a Google Maps API key from the [Google Cloud Console](https://console.cloud.google.com/).
   - Add your API key to the relevant configuration or script file where the Maps embed is loaded.
